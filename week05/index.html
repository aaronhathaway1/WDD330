Week 5 Notes

Chapter 8 Bugs and Errors

  1. Bugs are flaws in we write into our programs
  2. In JS, because of the looseness of the language, typos typically won't be caught
  3. You can enable strict mode in JS by putting "use strict" at the top of a file or function body
  4. Using this mode is helpful in finding some bugs
  5. Type conversion and consistency can be a problem
  6. Instead of manually testing our programs and checking after any change is made to our code, we can write a test program
  7. While test programs take a little longer to write, they become extremely helpful once written
  8. Some of the work has been done for us in the form of test runners
  9. A test runner is some software that help us build and run collections of tests
  10. Once a error has come to the forefront, next its cause must be identified
  11. Do not make random changes to your code to get it to change
  12. Think, analyze! Why is the program doing what it's doing
  13. Some errors will be outside of our control
  14. Instead of just crashing, it may be better to inform the user, or to keep moving forward
  15. We can also use try/catch statements as a way of anticipating and preventing failures
