Week 3 Notes

Chapter 6 Notes

  1. Large programs are often complex, causing programmers to be confused
  2. Confused programmers introduce bugs that have plenty of space to hide in
  3. Abstraction is talking about programs from a higher level, with out specifics
  4. As programmers we might too often work too low on the abstraction ladder
  5. We can use functions to do task repetively to save us time and complication
  6. Functions that operate on other functions are called higher order functions
  7. This allows us to abstract over actions
  8. We can also use functions to change other functions
  9. The ranges property conatins an arrat of Unicode characters, each is a two-element array
  10. The map function takes an array, modifies each element and takes the new element and puts it into an array
  11. The charCodeAt method gives us a code unit

Chapter 14 Notes

  1. DOM is short for Document Object Model
  2. A data structure is called a tree when it has a branching structure, no cycles and a single defined root
  3. Each DOM node object has a nodeType property
  4. You can access and therefore alter elements in the document
  5. There are many ways to access a node. Do not try to access by following the branches
  6. Typically its better to access through an id, class or an object
  7. There are many methods that allow for the changing of a node
    • remove
    • appendChild
    • insertBefore
    • replaceChild
  8. We create text nodes by "document.createTextNode"
  9. document.createElement
  10. Elements' attributes can be accessed through the DOM
  11. They can also be accesss through "getAttribute" and "setAttribute
  12. Some HTML elements take up the whole width of a page, these are called block elements
  13. Paragraphs and headings fall into this category
  14. Other elements are rendered on the same line. These are called in-line elements.
  15. We can alter an element's style through JS as well.
  16. CSS stands for Cascading Style Sheets
  17. Typically the most recently applied rule is applies when there are contradicting rules

Chapter 15 Notes

  1. There are many ways to register user input
  2. Actively monitor all input sources(keyboard, mouse, etc)
  3. Set up a queue where those input can go and wait to be processed
  4. Use an event handler
  5. We can addEventListener
  6. We can removeEventListener
  7. Event handler functions are passed an argument: the event object
  8. Events propagate outwards if there are handlers on both a parent and child node
  9. We can stop propagation with event.stopPropagation();
  10. JavaScript event handlers are called before default behaviors, typically
  11. We can prevent deafult behaviors by using event.preventDefault();
  12. Keys fire a keydown when pressed and a key up event when released
  13. There are two ways to point at stuff on screen, using a mouse or a touchscreen
  14. We can also use scroll events
  15. The focus and blur events do not propogate
  16. When a page finishes loading, a load event is fired
  17. We should use web workers that runs alognside the main script to do intensive, time consuming tasks
