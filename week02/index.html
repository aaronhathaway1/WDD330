Notes

Chapter 2 Notes

  1. You can bind a value to a name by using the "let" keyword.
  2. Values can be changed
  3. Bindings are more like tentacles than boxes, they grasp values
  4. Multiple bindings can be specified "Let one = 1, two = 2;"
  5. "var" and "const" also allow a user to create a binding
  6. The const keyword cannot be changed later. It is constant.
  7. The names of bindings can include digits but cannot start with one.
  8. The names of bindings can use '$' or '_' but no other special characters or punctuations.
  9. Functions are programs wrapped in a value.
  10. if/else statements allow for conditional execution/control flow
  11. You can use 'while' and 'do while' loops to make code more efficient
  12. For readability we indent segments of code that are part of a larger statement
  13. 'For while' loops can also be used.
  14. The 'break' and 'continue' keywords can be used to get out of a loop.
  15. Switch/Case works in JS as well. Don't forget the 'break' and the 'default'
  16. Good practice for naming bings is 'fuzzyLittleTurtle'

Chapter 3 Notes

  1. 'return' is a keyword that causes control to jump out of the current statement back to the code that called the function.
  2. Parameters passed into the function behave like regular bindings but are given their values by the caller.
  3. Bindings created inside functions can only be used in said functions. They have a scope of local.
  4. For bindings created outside of any function the scope is the whole function. These are global bindings.
  5. The 'var' keyword causes a binding to be accessible outside of its scope or block.
