Week 10 Notes

Client-Side Form Validation Notes

  1. Before submitting data to a server, it is important to ensure all required form controls are filled out
  2. Client-side validation should not be considered an exhaustive security measure
  3. Security measures should be in place on both the server side and the client side
  4. There are two types form validation
  5. built-in form and Javascript
  6. Form Controls include
  7. required
  8. minlength
  9. maxlength
  10. min and max
  11. type
  12. pattern
  13. The pattern attribute expects a regular expression

Using Fetch Notes

  1. The Fetch API provides a JS interface
  2. It provides global fetch()
  3. A fetch returns a promise
  4. A promise will reject a TypeError
  5. The Headers interface allows you to create your own headers
  6. Response instances are returned when fetch() promises are resolved
  7. The most common response properties you'll use are the following
  8. response.status
  9. response.statusText
  10. response.ok
