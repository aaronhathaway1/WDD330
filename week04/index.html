Week 4 Notes

Chapter 18 "HTTP and Forms" Notes

  1. HTTP(Hypertext Transfer Protocol") is the mechanism we use to request and provide data on the World Wide Web
  2. Common methods used in specifying resources are GET, DELETE, PUT, and POST
  3. Forms are how the user communicates with the website
  4. JS can make HTTP requests with fetch
  5. Calling fetch returns a promise
  6. This promise resolves to a Response object
  7. We use secure HTTP protocol for things that must remain secret or unmodified
  8. This looks like "https://"
  9. These are some common input tags
    • text - a single line of text
    • password - same as text but hides the text that is typed
    • checkbox - an on/off switch
    • radio - (Part of) a multiple choice field
    • file - allows the user to choose a file from their computer
  10. Form fields can get keyboard focus
  11. This means you must enter them by clicking in order for them to recieve input
  12. Form fields can be disabled as an attribute
  13. A form with a button with a type = "submit" will cause the form to be submitted when the button is pressed
  14. We can take information from forms and store it in localStorage

Chapter 6 "The Secret Life of Objects" Notes

  1. Object-Oriented programming is concerned with breaking large programs down into more manageable parts
  2. This concept is more accurately depicted as encapsulation
  3. We interface with objects through their methods and their properties
  4. Properties of the objects are public and private
  5. Methods are properties that hold function values
  6. A prototype is an object that is used as a fallback source of properties
  7. This is akin to oop's classes.
  8. This is called an instance in js
  9. We use a constructor function to make sure our instances have the correct properties
  10. Put the keyword 'new' in front of a function call to signify it as a constuctor
  11. A 'map' is a data structure that associates values(the keys) with other values.

Chapter 10 "Modules" Notes

  1. Modules are an attempt to specify which other pieces it relieas on
  2. The relations between modules are called dependecies
  3. It is a way of isolating code instead of it all being one big conglomerated file
  4. A package is a chunk of code that can distributed
  5. Packages normally come with documentation
  6. Some infrastructure is required for packages to be used, updated, etc
  7. This infrastructure is NPM
  8. NPM is an online repository of packages and a program that helps you download and install packages
  9. NPM combined with CommonJS modules was a good way for JS programmers to share good but we deserve better
  10. In 2015, ECMAScript was introduced to JS as the new module system
