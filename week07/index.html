Week 7 Notes

Chapter 11 Asynchronous Programming Notes

  1. Synchronous programming is when things happen one at a time.
  2. Asynchronous programming is when multiple things happen at the same time.
  3. One way of making synchronous programming faster it to start another thread of control
  4. Callback functions are used to implement asynchronous programming
  5. A promise is an asynchronous action that may complete at some point and produce a value
  6. We can us the "then" method to get the result of a promise
  7. To create a promise, you can use "Promise" as a constructor
  8. When working with networks, its a good idea to make our request function retry automatically before giving up
  9. Generally, it is a good idea to include a promise in our function
  10. Network flooding is the idea of sending messages thorughout the entire network
  11. This is done by one node receiving the message and sending it to its neighbors except the one who sent the orginal message
  12. This concept allows for messages to reach their destination even if the most immediate path is blocked
  13. Message routing is the idea of sending a message along the most direct route in order to cut down on useless data transfers
  14. A list of of the nests and their immediate neighbotrs is needed.
  15. This list is created and maintained by network flooding
  16. Generator functions can be paused and then resumed. They are similar to async functions but have no promises
  17. A function becomes a generator function by using this syntax when creating the function: function*
